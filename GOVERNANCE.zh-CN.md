# TraceArena 治理

[English](GOVERNANCE.md) | [简体中文](GOVERNANCE.zh-CN.md)

TraceArena 初期采用轻量、创始人主导的治理方式；项目负责人为张诺亚。

常规 bug 修复、测试、文档、示例和兼容性改进通过 Pull Request 审阅。公共契约、结算权威、安全边界、许可证或治理变更，应先在设计提案中讨论。

维护者负责守护通用运行时/场景边界，要求所有行为声明有证据，并以尊重的方式审阅贡献。商业用户不获得篡改客观结果、绕过安全审查或宣称官方认证的权力。

Apache-2.0 许可证不授予 TraceArena 商标、官方认证标识或暗示背书的权利。社区项目可以称为“built with TraceArena”；只有被授权项目可以使用“Official”或“TraceArena Certified”。
