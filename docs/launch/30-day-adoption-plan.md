# TraceArena 30-day adoption plan

This is an execution checklist for turning the public release into measurable
developer usage and qualified commercial conversations. It is a target plan,
not a claim about current traction.

## Days 1–3 — Make the first run unavoidable

- Publish the short English and Chinese launch copy in `tracearena-v0.1.3-launch.md`.
- Put the GitHub URL, Hugging Face demo URL, and one-command install command in
  every post.
- Ask readers to reply with the first world they would build; do not ask for a
  generic “please star”.
- Track: demo visits, repository visits, successful installs, and scenario ideas.

## Days 4–10 — Show one concrete trajectory

- Publish one annotated replay: observation → tool call → typed action → world
  transition → settlement.
- Record a five-minute screen capture using the full local frontend, with mock
  providers clearly labelled.
- Open one narrowly scoped contribution issue for a public scenario pack or
  verifier.
- Track: repeat visitors, forks, issue conversations, and completed local runs.

## Days 11–20 — Recruit builders

- Invite 5–10 agent, evaluation, or simulation developers to run the same
  fixture and critique the evidence/settlement contract.
- Offer a small “scenario office hour” and publish the resulting design notes.
- Accept only contributions with reproducible fixtures, settlement authority,
  and asset provenance.
- Track: external contributors, scenario-pack pull requests, and integrations.

## Days 21–30 — Convert qualified demand

- Publish a case-oriented post for one target domain (enterprise operations,
  code evaluation, governance, or research).
- Invite teams with a concrete workflow to request a paid scoping call.
- Use the commercial-support brief to qualify domain, agents/tools, evidence,
  deployment, timeline, and data-handling requirements.
- Track: qualified conversations, paid discovery proposals, and conversion to a
  scoped implementation.

## Weekly evidence review

Keep a small public or private ledger with these fields:

```text
date · channel · audience · link · action requested · result · next follow-up
```

Do not inflate stars or traffic. The strongest signal is a reproducible run,
an external scenario contribution, or a team willing to define a paid problem.
