## Lead handoff (private tracker reference only)

Do not put email addresses, customer data, credentials, or private run
archives in this file. For a qualified lead, record privately:

- source URL and consent to follow up;
- scenario and decision-maker/technical owner;
- evidence and acceptance criterion;
- expected start window and next action;
- whether the conversation is community support, pilot, or paid work.

A lead is qualified only when the conditions in
[OUTREACH_AND_FEEDBACK_PLAYBOOK.md](OUTREACH_AND_FEEDBACK_PLAYBOOK.md) are met.

## Reporting rules

- Report zero explicitly; do not replace missing data with estimates.
- Separate synthetic, historical, simulated, and live runs.
- Include the commit, fixture/seed, provider, and tool versions for replay
  claims.
- Never publish customer identity, private traces, API keys, or guaranteed
  business or investment outcomes.
