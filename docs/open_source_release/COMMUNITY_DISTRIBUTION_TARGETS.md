# Community distribution targets

Updated: 2026-07-18

This is a small, permission-aware distribution queue for TraceArena. It is
designed to produce useful technical feedback, not a burst of promotional
posts. Every public post must point to the same no-key Demo, Quickstart, and
feedback form.

## 1. Owned channels — publish first

- **GitHub Discussion #8:** [v0.1.4 technical discussion](https://github.com/tonyhyworld/TraceArena/discussions/8)
- **GitHub issue #7:** [design-partner pilot](https://github.com/tonyhyworld/TraceArena/issues/7)
- **HF Space:** [TraceArena Demo](https://tonyworld888-tracearena-demo.static.hf.space/index.html)

Post angle: show one reproducible run, one rejected or constrained action, and
one question about whether the evidence → action → event → settlement →
outcome trace is useful. Measure comments, completed first runs, and qualified
pilot requests separately.

## 2. Hacker News — one technical Show HN

Use the [Show HN guidelines](https://news.ycombinator.com/showhn.html): the
title must begin with `Show HN`, and the post should demonstrate something
people can try. Hacker News also cautions against using the site primarily for
promotion, so publish only after the Demo and Quickstart are healthy and be
ready to answer technical questions.

Suggested title:

> Show HN: TraceArena — auditable multi-agent worlds with replayable outcomes

Lead with the browser-local replay, deterministic fixture, and the separation
between agent decisions and world settlement. Do not lead with the paid pilot
or claim adoption, revenue, or benchmark superiority.

## 3. Hugging Face — Space-first distribution

Keep the [HF Space](https://huggingface.co/spaces/tonyworld888/tracearena-demo)
as the primary artifact. The [Spaces documentation](https://huggingface.co/docs/hub/en/spaces)
supports Spaces as shareable hosted demos; the Space README and footer already
link to the GitHub discussion, feedback form, and pilot issue.

Suggested post: explain the deterministic browser-local replay, its safety
boundaries, and how to reproduce the result. Do not imply that the replay is an
autonomous LLM benchmark.

## 4. Reddit and other communities — permission and contribution first

Do not cross-post the same launch copy. Read the destination community's
current self-promotion rules, participate in relevant technical discussions,
and ask moderators when required. The recent
[r/LocalLLaMA rule update](https://www.reddit.com/r/LocalLLaMA/comments/1su3ao4/rlocalllama_rule_updates/)
shows why a direct promotional post should wait until the account has useful
community participation.

## 5. Adjacent evaluation communities — contribute before announcing

- [AgentEval.org](https://www.agenteval.org/) describes itself as an open,
  community-driven initiative for domain-specific benchmarks and evaluation
  methods. Start with a technical comparison or scenario contribution; do not
  assume a project-post channel exists.
- [AI Agent Engineering](https://ai-agent-engineering.org/) is a community
  space for agent builders. Read its current participation and submission
  rules before proposing a walkthrough.
- [GitHub discussions on agent evaluation](https://github.com/karpathy/autoresearch/discussions/406)
  show the kind of concrete question that attracts useful technical exchange:
  how to measure why an agent succeeds or fails, not merely which model wins.

## Tracking and stop conditions

For each channel record the post URL, date, unique visitors, first successful
run, meaningful technical replies, scenario contributions, and qualified pilot
requests. Stop or revise a channel when it creates no meaningful interaction
after one carefully targeted post; do not increase volume to compensate.

No channel may be used to fabricate users, stars, comments, revenue, customer
logos, or performance claims.
