# AI World 世界路线图

这份路线图把“任何人都可以加载世界”落实为可核验的生态入口。只有已经在仓库中可运行、可测试的内容才标记为 **Available**；其他内容是公开征集方向，不代表已实现、已接入真实数据或已有合作方。

## Available now

### Synthetic Market Replay

- 类型：合成、确定性、离线世界
- 目标：在同一组市场证据和风险边界下，让不同策略完成结构化决策并由模拟账本结算
- 入口：[Market Replay](../../examples/market_replay/README.md)
- 约束：无 API Key、无券商连接、无真实订单，不构成投资建议

### Incident Response World

- 类型：合成、确定性、非金融世界
- 目标：在事件确认、缓解、健康检查和恢复之间按依赖关系完成服务恢复
- 入口：[Incident Response World](../../examples/incident_response_world/README.md)
- 可观察结果：世界会拒绝没有通过健康检查的提前恢复声明，并记录拒绝原因

## Community directions

这些方向需要贡献者先用合成或公开许可数据定义一个最小世界，再讨论是否接入外部事实：

| 世界方向 | 可先定义的目标 | 第一批资源与规则 | 推荐贡献物 |
| --- | --- | --- | --- |
| 城市治理 | 在预算和公平约束下选择一组公共服务方案 | 人口分区、预算、服务容量、最低公平阈值 | 场景包、政策结算器、回放视图 |
| 医药研发 | 在证据和实验预算约束下比较候选研究路径 | 假设、实验成本、结果置信度、停止规则 | 合成实验世界、证据规则、失败案例 |
| 物流与能源 | 在时间、库存或供需约束下完成配送/调度目标 | 资源、网络、时钟、容量和违约后果 | 工具适配器、确定性验证器、指标 |
| 机器人与现场作业 | 在权限和安全边界内完成连续任务 | 环境状态、动作空间、安全检查和人工确认点 | 模拟器适配、动作契约、可视化 |

## 如何把方向变成世界

1. 先提交一个合成、可复现的 [场景包提案](https://github.com/tonyhyworld/TraceArena/issues/2)。
2. 明确一个目标、有限资源、角色权限、可验证规则和权威结果。
3. 让至少一个行动被接受、一个行动被拒绝，并在回放中解释原因。
4. 为同一 fixture 提供确定性重放、测试和限制说明。
5. 通过 PR 后，再讨论公开数据、外部工具或私有试点的接入边界。

路线图不承诺所有方向都会由维护者实现；最有价值的贡献是一个小而真实、规则清楚、结果可复现的世界。
