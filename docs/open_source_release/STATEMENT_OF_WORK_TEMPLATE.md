## 3. 交付物

1. 一个经双方确认的场景契约、fixture 和验收说明；
2. 2–4 个 Agent/模型配置的同口径运行矩阵；
3. 至少 10 次受控、可复现的 evaluation runs；
4. evidence → action → event → settlement → outcome 证据链导出；
5. 成功/失败分类和差异路径报告；
6. 一次不超过 90 分钟的结果复盘会议；
7. `[约定的源码、配置、报告和运行工件归属]`。

## 4. 验收标准

- 同一版本、fixture 和配置可重放，并产生一致的 `deterministic_replay_sha256`；

- 关键结论可以回链到运行工件；
- 失败可以定位到约定的责任层（研究、规划、工具、权限、规则或结果）；
