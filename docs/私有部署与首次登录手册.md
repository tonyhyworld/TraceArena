# TraceArena 私有部署与首次登录手册

本文面向在服务器、内网或企业环境中部署完整 TraceArena / AI World 的管理员。完整前端包含观众端、运营台、用户权限、模型配置、场景管理和实时 WebSocket，因此部署完成后默认需要登录。

## 1. 部署前准备

建议准备 Python 3.10+、Node.js 18+、可持久化磁盘，以及生产环境使用的域名和 HTTPS 反向代理。

在项目根目录执行一键安装：

```bash
./scripts/install.sh
```

Windows PowerShell：

```powershell
.\scripts\install.ps1
```

## 2. 配置后端环境

```bash
cd backend
cp .env.example .env
```

生产环境应配置并长期保存以下密钥：

```dotenv
AIWORLD_JWT_SECRET=请填入长期保存的随机密钥
AIWORLD_SECRET_KEY=请填入长期保存的Fernet密钥
```

如果留空，系统首次启动会自动生成并写回 `backend/.env`。更换 JWT 密钥会使已有登录 token 失效；丢失 BYOK 加密密钥会导致已保存的用户 API Key 无法解密。

模型 Provider 的 API Key 按需配置，例如 `DEEPSEEK_API_KEY`、`MINIMAX_API_KEY` 或 `HF_TOKEN`。只验证无 Key 回放时可以全部留空。

## 3. 首次创建管理员账号

系统没有默认账号，也没有自助注册。首次部署必须在服务器上手动创建至少一个管理员账号：

```bash
cd backend
source ../.venv/bin/activate
python scripts/create_user.py
```

脚本会询问用户名、显示名、是否管理员，以及密码和确认密码。首次账号请选择管理员；用户名只能使用字母、数字、下划线或横杠，长度 3–32 位；密码至少 6 位，生产环境建议使用密码管理器生成高强度密码。

账号默认写入：

```text
backend/user_data/users.json
```

如果使用独立数据盘，可在启动前设置：

```bash
export AIWORLD_USER_DATA_ROOT=/srv/tracearena/user_data
```

## 4. 启动和登录

后端示例：

```bash
cd backend
source ../.venv/bin/activate
AIWORLD_CONFIG=./framework.capital_market.yaml \
PYTHONPATH=. \
python -m uvicorn app.main:app --host 127.0.0.1 --port 8001
```

开发环境前端：

```bash
cd frontend
npm run dev
```

打开 `http://localhost:5173`，使用刚刚创建的管理员账号登录。管理员登录后可以在运营台的“用户管理”中创建普通用户并分配权限。

生产环境构建前端：

```bash
cd frontend
npm run build
```

使用 Nginx、Caddy 或企业网关托管 `frontend/dist/`，并将后端 API 和 WebSocket 请求转发到 `127.0.0.1:8001`。生产环境必须启用 HTTPS，并正确转发 WebSocket 升级请求。

## 5. 必须持久化的内容

容器、临时磁盘或自动扩缩容环境必须把以下内容挂载到持久化卷：

```text
backend/.env                 # JWT 和 BYOK 加密密钥
backend/user_data/           # 用户、权限、用户 Key 和用户运行数据
backend/runs/                # 运行日志、回放和审计结果
```

丢失 `user_data/` 会丢失账号和用户级数据；丢失 `.env` 中的密钥会导致 token 或已加密 API Key 失效。建议定期备份，并限制备份文件的访问权限。

## 6. 用户和权限管理

管理员可以创建普通用户、分配观众端/运营台/对局控制/模型配置/场景管理/基准测试/数据导出权限，并可修改显示名或重置普通用户密码。管理员账号不能在运营台删除。用户可以在个人账户中心修改自己的密码，用户运行数据按 `user_id` 隔离。

## 7. 健康检查和排查

```bash
curl http://127.0.0.1:8001/health
```

| 现象 | 处理方式 |
|---|---|
| 用户名或密码错误 | 确认当前实例的 `user_data/users.json` 中存在该账号 |
| 重启后无法登录 | 检查 JWT 密钥和 `user_data/` 是否持久化 |
| 页面打开但实时对局不动 | 检查反向代理的 WebSocket 转发及 HTTPS/WSS 配置 |
| 运营台菜单为空 | 检查账号是否为管理员或是否已授予对应权限 |
| 场景加载失败 | 检查 `AIWORLD_CONFIG`、场景路径和后端日志 |

## 8. 安全检查清单

- [ ] 已创建管理员账号并使用强密码；
- [ ] 已持久化 `.env`、`user_data/` 和必要的 `runs/`；
- [ ] 生产环境已启用 HTTPS 和反向代理；
- [ ] 未把密码、API Key、客户数据或私有运行归档提交到 Git；
- [ ] 已验证登录、观众端、运营台和 WebSocket 实时对局；
- [ ] 已验证 `/health` 返回正常；
- [ ] 已配置备份和恢复流程。
