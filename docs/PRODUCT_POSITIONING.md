# AI World / TraceArena 产品定位（权威稿）

> 状态：2026-07-21 定稿。GitHub README、官网、Product Hunt、视频和商业介绍均应以本文为准。

## 一句话

**Define a world. Let agents compete. Watch the path to the outcome.**
**定义一个世界，让 Agent 在其中竞争，并看见答案如何产生。**

## 产品定义

**AI World 是一个面向不同领域场景的多智能体问题求解与可视化演绎系统。**

用户定义目标、资源、约束、工具、结果标准以及“世界如何反馈”，加载一个场景包，让多个 Agent 在同一个受约束的世界中持续行动、竞争和试错；系统负责推进世界变化、比较候选路径、结算结果，并将完整决策链路可视化、可回放地呈现出来。

AI World 不要求每个场景都拥有真实物理反馈。不同领域可以选择最适合自身问题的世界模型：专家规则、确定性算法、训练后的 World Agent、专业模拟器、真实系统，或上述方式的混合。关键不在于所有结果都来自物理世界，而在于世界模型被明确声明、版本固定、边界可见、过程可审计，并且其可信程度不被夸大。

## 旗舰市场入口：资本市场专业评测

对外介绍采用“两层定位”：**资本市场专业评测是最先被用户理解的旗舰世界，AI World OS 是支撑它并可扩展到其他领域的底层平台。** 首屏应先回答投资 Agent 如何被公平、专业、可审计地比较，再说明场景包与通用运行时。

资本市场场景的核心表达必须包括：

1. 不同投资 Agent 使用相同本金、证据边界、工具权限、市场时钟和成交规则；
2. 研究来源、新鲜度、工具调用和行动证据均被记录；
3. 买入、卖出、等待均为可校验的结构化行动；
4. 模拟账本统一计入佣金、滑点与持仓变化；
5. 同时比较收益、基准超额收益、最大回撤、换手率与交易成本；
6. 全链路可回放，不连接券商，不提供投资建议。

这不是把 TraceArena 收缩为投资产品，而是用一个高信息密度、规则明确且结果可量化的专业世界证明通用运行时的价值。

## 品牌分工

| 名称 | 角色 |
|---|---|
| **AI World** | 产品理念与系统形态：把现实问题变成可运行、可观看的 AI 世界 |
| **TraceArena** | 开源运行时与实现：场景加载、多 Agent 调度、结算、追踪与回放 |
| **场景包** | 具体世界：资本市场、城市治理、医药研发、物流、机器人等 |
| **可视化演绎** | 用户真正看到的核心体验：过程、竞争、代价与结果 |

## 核心价值

1. 把现实问题转化为可运行的 AI 场景。
2. 让多个 Agent 在同一约束下寻找不同解决路径。
3. 让领域专家选择合适的世界反馈方式，而不是被迫从零开发完整物理模拟器。
4. 对比不同路径的过程、代价、风险和最终结果。
5. 让领域专家无需从零开发完整模拟器，就能创建自己的 AI World。
6. 通过场景包形成可持续扩展的世界生态。

## 系统链路

```text
场景包定义
  ↓
目标、资源、约束、工具、世界模型
  ↓
多 Agent 行动与竞争
  ↓
世界事件与后果
  ↓
权威结算与评价
  ↓
过程可视化与回放
```

## 统一世界模型架构

场景包通过同一个 World Adapter 契约接入六类世界模型实现：

| 世界模型实现 | 适用情形 | 结果边界 |
|---|---|---|
| 专家规则 `rule_based` | 规则清楚、需要快速建模的专业流程 | 代表专家假设，不等于客观物理规律 |
| 确定性算法 `algorithmic` | 优化、排程、账本、可计算约束 | 在输入与算法边界内可复现 |
| 训练模型 / World Agent `learned` | 规则难以穷举、存在经验判断 | 必须声明模型版本、置信度、验证集与局限 |
| 专业模拟器 `simulator` | 电网、交通、能源、机器人等已有环境 | 可信度取决于模拟器假设与校准范围 |
| 真实系统 `reality` | 可安全读取或执行的现实业务系统 | 必须保留来源、权限、回执与时间戳 |
| 混合模型 `hybrid` | 真实数据 + 规则/算法/模型/模拟器 | 必须标明每一段状态变化的来源 |

这六种实现回答“世界如何变化”；现有四种结算类型回答“结果由谁证明”。二者彼此正交，不能混为一谈。

## 它不是

- 普通聊天机器人；
- 单纯的 Agent 编排框架；
- 只提供最终答案的模型应用；
- 只为沉淀训练数据的平台；
- 默认直接控制真实物理设备的自动驾驶系统。

## 展示原则

产品展示应从「介绍系统架构」转向「展示一个场景如何被加载、运行、竞争、结算和可视化」。

优先顺序：

1. 加载场景包；
2. Agent 在约束下行动与竞争；
3. 世界验证与权威结算；
4. 观众端看见路径与结果；
5. 回放与对比。

轨迹数据、评测与商业服务是副产品与扩展能力，不是首句定位。
