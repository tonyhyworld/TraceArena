# Submission Notes

## Before submitting

1. Confirm the author list, affiliations, and corresponding author email.
2. Replace any product-facing wording with the final academic terminology preferred by the authors.
3. Run the evaluation protocol described in the paper and commit the fixtures, logs, and analysis scripts.
4. Add exact version, commit hash, hardware, model, prompt, and cost information.
5. Check third-party licenses for every scenario asset used in figures or experiments.

## Recommended public-first path

1. Publish the paper and artifacts in this repository.
2. Create a versioned release and archive it on Zenodo for a DOI.
3. Upload the final PDF and source package to arXiv after author and category confirmation.
4. Submit the extended version to a multi-agent systems, agent evaluation, AI simulation, or applied AI venue.

## Evidence policy

Do not report a benchmark score, success rate, business outcome, or capability claim unless it is generated by a reproducible experiment and included as an artifact. The current paper deliberately separates implemented capabilities from future evaluation hypotheses.
