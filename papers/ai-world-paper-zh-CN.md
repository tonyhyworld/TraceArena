# AI World：面向领域专家的可验证多智能体问题求解框架

## 摘要

大语言模型正在从对话系统走向 Agent，但大量 Agent 应用仍停留在一次性回答或固定工作流阶段。这使领域专家难以表达自己的专业约束，使多个 Agent 难以在同一问题上探索竞争路径，也使用户难以验证最终结果是如何产生的。本文介绍 **AI World**，以及其开源实现 **TraceArena**：一个面向领域专家的场景包框架，用于把专业问题转化为可运行的 AI 世界。场景包声明目标、角色、资源、规则、工具、外部数据、终止条件、结算逻辑和展示元数据。世界加载后，多个 Agent 在受约束的观察—规划—行动—反馈循环中持续执行；世界内核验证动作并推进状态；结算运行时通过模拟规则、外部事实、确定性验证器或混合方式计算结果；事件账本保存可审计轨迹，并支持可视化回放。本文的核心贡献是建立领域知识与 Agent 执行之间的可复用接口：领域专家负责定义世界，运行时负责调度、验证、结算和可观察性。本文描述系统架构、执行协议、场景包设计和比较不同模型与策略的评估方法，并随开源仓库提供示例与复现说明。

**关键词：** 多智能体系统；Agent 环境；场景包；Agent Loop；可验证执行；AI 仿真；决策可追溯；可视化回放

## 1. 引言

一个能够生成合理段落的 Agent，不一定能够解决真实问题。真实决策通常会持续一段时间，消耗资源，受到权限和规则约束，与其他决策者互动，并在较晚阶段产生后果。领域专家还需要明确：哪些行动是合法的，什么样的结果才算好。这些要求很难由一次提示词、一次性基准测试或固定工作流引擎完成。

AI World 通过“可运行世界”抽象解决这一问题。世界不被写死在操作系统中，而由场景包定义。领域专家声明目标、参与者、资源、规则、工具、证据来源和结算标准；多个 Agent 进入同一个世界后，可以持续观察、行动、竞争或协作、接收反馈、修正计划，直到目标或终止条件触发。运行时把完整链路呈现出来：Agent 看到了什么、做出了什么判断、尝试了什么动作、世界如何反馈，以及最终结果依据哪些事实被结算。

本文的中心观点是：

> 领域专家应该能够在不先开发完整 Agent 平台的情况下，把一个真实问题转化为可运行的 AI 世界，并让重要行动保持可验证、可回放。

本文的主要贡献包括：

1. 提出将领域定义与通用运行时分离的场景包抽象；
2. 提供带有世界校验、反馈、有限重试和终止条件的多智能体执行循环；
3. 提供支持模拟、外部现实、确定性验证和混合世界的结算与轨迹模型；
4. 提供将可信事实转换为可读、可回放决策故事的可视化链路，同时保证叙事组件不能成为世界事实的权威来源。

## 2. 设计目标与非目标

### 2.1 设计目标

AI World 的设计目标是：

- 降低领域专家构建 Agent 环境的门槛；
- 支持持续行动，而不是只完成一次性回答；
- 在相同世界约束下比较多个 Agent 和多种策略；
- 让世界状态、动作、事件、结算和轨迹可审计；
- 支持不同 LLM Provider 以及内部或外部工具；
- 通过实时可视化和回放，让复杂执行过程可理解。

### 2.2 非目标

AI World 不声称 LLM 在任何领域都正确、安全或可以完全自主。它不能替代生产控制、监管审查、人工授权或物理安全系统，也不以训练数据交易平台为首要定位。轨迹和评估记录是有价值的输出，但产品的首要价值是提供一个可复用的专业世界框架，用于解决、观察和复盘领域问题。

## 3. 世界模型

设场景包定义一个世界 (W)，其状态为 (s_t)，Agent 集合为 (A)，动作集合为 (U)，工具集合为 (T)，观察集合为 (O)，结算函数为 (S)。在第 (t) 步，Agent (a_i) 获得经过权限过滤的观察 (o_{i,t}=V_i(s_t))，其中 (V_i) 是该角色的可见性策略。Agent 产生计划和候选动作 (u_{i,t})。世界内核使用合法性函数 (L(u_{i,t},s_t)) 进行验证。若动作合法，状态转移函数产生 (s_{t+1}) 和世界事件 (e_t)；否则返回结构化失败反馈，供 Agent 反思和修正。

当目标谓词、终止谓词或时间/资源预算达到时，一次运行结束。结算运行时根据 (r=S(s_{0:T},e_{0:T},x)) 计算结果，其中 (x) 可以包括市场价格等外部事实或确定性验证器输出。轨迹账本保存观察、模型决策、工具调用、动作、失败、事件、结算记录和展示事实。

## 4. 系统架构

```text
场景包
  目标 / 角色 / 资源 / 规则 / 工具 / 结算 / 展示
                              |
                              v
TraceArena 运行时
  加载 -> 调度 -> Agent Harness -> 世界内核 -> 事件账本
                                  |              |
                                  v              v
                              LLM Provider    结算运行时
                                  \              /
                                   v            v
                                可信事实与轨迹
                                      |
                                      v
                            导演 -> 渲染 -> 回放 / 运营台
```

### 4.1 场景包层

场景包是系统的扩展边界，声明世界词汇、角色、动作、资源、工具、时钟、目标、结算清单和渲染绑定。运行时不假定所有世界拥有相同的动作结构或奖励函数。

### 4.2 Agent Harness

Harness 将 LLM 约束为有边界的行动者：组装角色提示和世界观察，暴露经过许可的能力，调用工具，解析结构化动作，接收反馈，并控制重试与预算。失败动作不会被静默视为成功，而会成为明确的反馈事件。

### 4.3 世界内核

世界内核是状态转移的权威组件，负责校验权限、动作结构、资源可用性、可见性、阶段规则并创建事件。这一隔离避免语言模型或叙事组件自行宣布结果。

### 4.4 结算运行时

不同领域需要不同事实来源，AI World 支持四类结算模式：

| 模式 | 结果来源 | 示例 |
|---|---|---|
| simulation | 场景规则 | 影响力变化、资源变化 |
| external_reality | 外部数据或服务 | 观测价格、外部任务结果 |
| deterministic_verifier | 可复现校验器 | 测试、Schema、精确判卷 |
| hybrid | 外部事实与确定性规则组合 | 市场价格与组合账本 |

### 4.5 轨迹账本与回放

账本记录解释一次运行所需的事实。运行可以按时间线检查，也可以基于固定 fixture 重新回放，从而支持调试、模型对比、专家复核和后续实验分析。

### 4.6 导演与渲染器

导演将经过验证的事件和结算记录转换为观众能理解的描述和渲染指令。它可以解释一个动作为什么重要，但不能替代世界内核，也不能修改结算记录。这使叙事清晰度与事实权威相互分离。

## 5. Agent Loop 协议

标准循环为：

```text
观察 -> 规划 -> 选择能力 -> 执行 -> 校验
  ^                                  |
  |---------- 反馈 / 反思 ------------|
```

循环由明确策略约束：最大步数或 tick、Token 和工具预算、动作权限、世界终止条件、失败重试策略，以及必要时的人类审批门。因而“持续执行”指持续的目标导向工作，而不是无边界或不受控制的自动运行。

## 6. 如何创建领域场景包

领域团队可以分五步创建场景包：

1. **描述问题。** 定义可度量目标及完成条件；
2. **建模世界。** 明确参与者、资源、时间、信息、约束和外部依赖；
3. **声明动作和工具。** 规定合法动作、角色工具以及需要授权的副作用；
4. **定义结算。** 选择模拟规则、外部事实、确定性校验器或混合方式；
5. **定义展示。** 选择专家和观众需要看到的事件、指标、决策和对比。

同一个场景包可以在固定世界定义下替换模型、提示词、随机种子、角色策略和工具配置，重复运行并比较结果。

## 7. 应用方向

- **资本市场：** 在行情、风险预算和交易规则约束下比较研究与组合策略；
- **城市治理：** 推演交通、应急响应和公共资源配置政策；
- **医药研发：** 在预算、证据和试验阶段约束下规划实验；
- **物流供应链：** 协调订单、库存、路线、天气和突发中断；
- **制造与机器人：** 在受控世界中安排生产、处理故障并遵守安全边界；
- **教育与科研：** 构建用于教学、假设验证和模型评测的可复现实验世界。

## 8. 评估方法

本文刻意区分“已实现能力”和“已测量结果”。可复现实验应回答：

- 显式世界校验能否减少非法或不可能动作；
- 多 Agent 竞争能否比单 Agent 发现更多或更优路径；
- 反馈和反思能否提升失败后的恢复能力；
- 可视化回放能否降低专家定位决策原因的时间；
- 不同模型、随机种子和重复运行下结果是否稳定。

建议至少比较：单 Agent 与多 Agent、一次性回答与 Agent Loop、无工具与授权工具、无反思与反馈触发反思、叙事判断与权威结算。

建议指标包括目标完成率、结算得分、资源成本、规则违反率、失败恢复率、工具调用次数、重复运行方差、轨迹完整性和专家复核时间。所有数值都应附场景包版本、运行时 commit、模型和 Provider、提示词、工具配置、预算、随机种子、硬件以及原始回放材料。

## 9. 局限与风险

世界的质量取决于规则、数据、工具和结算设计。定义不充分的场景可能产生“看起来精确但实际上无效”的结论。外部数据可能延迟、不可用或受到授权限制。LLM 行为具有概率性，并可能受提示词和 Provider 变化影响。对于高影响领域，AI World 应作为分析、仿真和决策支持工具，必须配合相应的人类监督，而不能作为未经复核的权威。

## 10. 开源发布

TraceArena 是 AI World 的开源运行时。公开仓库包含场景包模板、可运行示例、本地回放路径、部署文档和贡献指南。社区可以通过贡献领域场景包扩展系统，而不必为每一个新领域修改核心运行时。

## 11. 结论

AI World 建立了领域知识与多智能体执行之间的实用接口。专业团队可以定义一个世界，将它加载到通用运行时，让 Agent 在明确约束下行动和竞争，并观察通往结果的完整路径。系统的核心对象不是一段对话答案，而是一条可验证的运行记录：观察、决策、动作、反馈、事件和结算可以被观看、回放、比较和改进。

长期来看，AI World 的价值来自建立在同一开源运行时之上的领域世界生态。金融专家、城市规划者、医学研究者和物流运营者，都可以贡献一个让 AI 为该领域重要问题工作的世界。
