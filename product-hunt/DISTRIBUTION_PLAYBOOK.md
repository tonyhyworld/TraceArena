# AI World OS distribution playbook

This playbook keeps every launch channel focused on one idea: **TraceArena is an open-source operating system for loading and running real-world AI worlds.**

## One-sentence positioning

Define a world's goal, resources, rules and tools; let competing agents search for a path; inspect every decision and settlement.

## Channel copy

### X / LinkedIn

> What if agents were evaluated inside a world instead of inside a chat window?
>
> TraceArena is an open-source AI World OS. Load a scenario pack, define goals/resources/rules, connect an LLM, and let agents compete over real constraints. Every observation, action, consequence and settlement is visible and replayable.
>
> Try the no-key replay: https://tonyhyworld.github.io/TraceArena
>
### Hacker News / Reddit

> We built TraceArena because a chat transcript is not a world. It is an open-source runtime for defining a goal, resources, rules and tools, then running multiple agents against the same constraints. The runtime records the full decision chain and authoritative outcome, so a result can be inspected and replayed. The first example is synthetic and requires no API key; contributors can add scenario packs for markets, logistics, governance, science or robotics.

## Launch sequence

1. Publish the Product Hunt launch only after the maker account, thumbnail and first gallery image are verified.
2. Reply to every early comment with a concrete replay link or a small scenario-pack example.
3. Within 24 hours, publish one short screen recording showing `define → observe → compete → act → settle → replay`.
4. Within 72 hours, open one narrowly scoped “build a world” issue and invite contributors to submit scenario packs.
5. Record source clicks, replay starts, GitHub stars, scenario-pack submissions and paid-support enquiries separately.

## Conversion path

`Product Hunt / social post → no-key replay → GitHub quickstart → scenario-pack contribution → commercial support`

The public launch should remain free and Apache-2.0. Revenue comes from optional implementation, private scenario design, deployment and domain-specific evaluation support; do not gate the first replay behind a sale.

## Call for contributors

Ask contributors to submit a small, reproducible world with:

- a measurable goal;
- explicit resources and rules;
- an authoritative settlement/evaluation function;
- a replayable example and provenance notes;
- no private data or undisclosed third-party assets.
